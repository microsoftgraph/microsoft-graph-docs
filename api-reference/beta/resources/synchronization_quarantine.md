# synchronizationQuarantine resource type

Provides information about [synchronizationJob](synchronization_job.md)'s quarantine state.

## Properties
| Property	   | Type	|Description|
|:---------------|:--------|:----------|
|currentBegan|DateTimeOffset|Date and time when quarantine was last evaluated and imposed. The Timestamp type represents date and time information using ISO 8601 format and is always in UTC time. For example, midnight UTC on Jan 1, 2014 would look like this: `'2014-01-01T00:00:00Z'`.|
|nextAttempt|DateTimeOffset|Date and time when next attempt to re-evaluate the quarantine will be made. The Timestamp type represents date and time information using ISO 8601 format and is always in UTC time. For example, midnight UTC on Jan 1, 2014 would look like this: `'2014-01-01T00:00:00Z'`.|
|reason|String| Code signifying why the quarantine was imposed. Possible values are: `EncounteredBaseEscrowThreshold`, `EncounteredTotalEscrowThreshold`, `EncounteredEscrowProportionThreshold`, `EncounteredQuarantineException`, `Unknown`.|
|seriesBegan|DateTimeOffset|Date and time when the quarantine was first imposed in this series (Series starts when quarantine is first imposed, and is reset as soon as quarantine is lifted). The Timestamp type represents date and time information using ISO 8601 format and is always in UTC time. For example, midnight UTC on Jan 1, 2014 would look like this: `'2014-01-01T00:00:00Z'`.|
|seriesCount|Int64|Number of times in this series quarantine was re-evaluated and left in effect (Series starts when quarantine is first imposed, and is reset as soon as quarantine is lifted).|
